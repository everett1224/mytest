说明：

关于编译器的函数的命名规则其实不但是要看编译器本身。
还要看声明。
这从另一个方面说明了声明的重要性！！！


C编译器-》print 编译成--------》print

CPP编译器（不使用extern “C”）-》 print 编译成了 ----------》_Z5printi

CPP编译器（使用extern “C”）-》 print 编译成了 ----------》print


如果使用-c选项，编译成的.o文件其实根本没有区别（如果不考虑函数名称的问题），
而真正的问题在于链接器上，
明显的G++的链接器链接的范围更广，例如可以识别iostream中的符号。
而GCC 的链接器的范围就不能找到。
当然，如果使用LD命令手动链接的时候，我没有做过实验，但是感觉，只要把包含的足够多，根本没问题。
